# PEDAGOGICAL_CANON_v1

## Purpose

This document defines the canonical pedagogical foundations of TeoryX.

Its purpose is to ensure that curriculum analysis, instructional planning, assessment design, lesson generation, tutoring behavior, and future AI-driven educational decisions remain coherent, evidence-based, and traceable.

This canon establishes:

* Authoritative pedagogical sources
* Source precedence
* Domain model ownership
* Concept mapping
* Conflict resolution rules

---

# Source Hierarchy

When pedagogical sources disagree, precedence is:

1. Jurisdiction Curriculum Requirements
2. TeoryX Pedagogical Canon
3. School Configuration
4. AI Synthesis

AI-generated reasoning may not override canonical pedagogical rules.

---

# Pedagogical Field Ownership Rule

Every pedagogical model and field in TeoryX must have a canonical source owner.

No pedagogical model, field, or AI-generated pedagogical recommendation may become part of the TeoryX domain unless it has an explicitly declared canonical source owner.

This rule applies to:

* Existing pedagogical fields
* Future pedagogical fields
* Future pedagogical domain models
* AI-generated pedagogical recommendations
* Validation rules
* Audit and provenance records

Purpose:

* Prevent ad hoc AI-generated pedagogy
* Ensure pedagogical traceability
* Support future auditability and validation
* Keep curriculum, pedagogy, assessment, tutoring, and intervention decisions grounded in approved sources

## Governance Requirements

When a new pedagogical model or field is introduced, its design documentation must declare:

* Canonical source owner
* Domain owner
* Intended use
* Validation rule
* Conflict resolution rule
* Audit/provenance expectation

AI synthesis may fill, adapt, or explain pedagogical fields only inside the constraints of the declared canonical source owner. AI may not invent new pedagogical categories, models, or instructional policies without human approval and source-owner declaration.

If a field combines multiple sources, the implementation and documentation must identify the source role clearly. For example, curriculum may define what skill is required, while How Learning Works may define how prerequisite knowledge or misconceptions are analyzed.

If sources conflict, the Source Hierarchy and Conflict Resolution rules in this canon apply.

* Jay McTighe

# Conflict Resolution

When sources disagree:

1. Curriculum requirements win.
2. Understanding by Design governs instructional architecture.
3. Learning science governs cognitive decisions.
4. Assessment evidence overrides instructional preference.
5. AI synthesis cannot override canonical sources.
