# TeoryX - UbD Summary for Analyst (CC)

## Source

Understanding by Design (UbD)

Chapters:

* Backward Design
* Understanding Understanding
* Thinking Like an Assessor

---

# Principle 1: Begin With The End In Mind

No diseñar contenido.

Diseñar resultados.

La pregunta correcta NO es:

"What should we teach?"

La pregunta correcta es:

"What should the student be able to understand and do?"

---

# Implication For TeoryX

La IA nunca debe comenzar generando una lección.

Debe comenzar generando:

1. Learning Objective
2. Desired Understanding
3. Evidence Of Understanding
4. Learning Experience

Orden obligatorio:

Learning Objective
↓
Assessment
↓
Lesson
↓
Tutor Interaction

No al revés.

---

# Principle 2: Understanding Is Different From Memorization

Un estudiante puede recordar información sin comprenderla.

Comprensión implica:

* explicar
* aplicar
* interpretar
* conectar ideas
* resolver problemas

---

# Implication For TeoryX

El progreso académico no debe medirse por:

* tiempo conectado
* páginas leídas

Debe medirse por:

* capacidad de aplicar conocimiento
* desempeño en evaluaciones
* capacidad de explicar conceptos

---

# Principle 3: Assessment Drives Learning

La evaluación define qué evidencia aceptamos como prueba de comprensión.

---

# Implication For TeoryX

Cada lección debe producir:

Assessment Blueprint

Conteniendo:

* learning objective
* success criteria
* mastery threshold

Ejemplo:

Objective:
Understand fractions.

Evidence:
Student solves real-world fraction problems.

Mastery:
80% score.

---

# Principle 4: Curriculum Is A Path

Una lección aislada tiene poco valor.

Debe existir una secuencia.

Grade
↓
Subject
↓
Unit
↓
Lesson
↓
Assessment

---

# Principle 5: Big Ideas Matter

El aprendizaje debe organizarse alrededor de conceptos importantes.

No alrededor de datos aislados.

---

# TeoryX Rule

Cada Lesson debe contener:

* Big Idea
* Essential Question
* Learning Objective

Ejemplo:

Big Idea:
Fractions represent parts of a whole.

Essential Question:
How can fractions help us describe the world?

Learning Objective:
Represent fractions visually.

---

# Final Recommendation

Toda generación de contenido IA debe seguir:

Stage 1:
Desired Results

Stage 2:
Assessment Design

Stage 3:
Lesson Generation

Nunca generar directamente una lección sin pasar por las etapas anteriores.
